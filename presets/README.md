# Presets

Промпты для `analyzetg analyze --preset <name>`. Каждый `.md` — один пресет.

Файлы, начинающиеся с `_` (например `_reduce.md`), пресетами не считаются —
это вспомогательные промпты для map-reduce.

## Доступные пресеты

| Пресет | Что выдаёт | Когда пригодится |
|---|---|---|
| `summary` | Короткое «что нового и неочевидного»: главное, идеи/решения, что посмотреть | Быстро понять, стоит ли читать чат глубже — дефолтный вариант |
| `digest` | Короткий пронумерованный список тем, 1–2 строки на каждую | Быстрый сканирующий дайджест |
| `action_items` | Markdown-таблица задач: кто/что/срок/статус/ссылка | Найти, что нужно сделать после чата |
| `decisions` | Markdown-таблица решений: решение/кто/когда/обоснование/ссылка | «Что мы тут договорились?» |
| `highlights` | 5–15 самых ценных сообщений, отсортированных по важности | «Не читать всё — покажи жемчуг» |
| `questions` | Таблица открытых вопросов (без ответа / частично / без консенсуса) | Вернуться и ответить/обсудить |
| `quotes` | Дословные памятные цитаты с автором и ссылкой | «Хорошо сказано» — сохранить |
| `links` | Внешние URL из чата, сгруппированные по темам | Собрать ссылки на инструменты/статьи |

Все пресеты выдают ссылки на конкретные сообщения вида
`[#12345](https://t.me/username/12345)` (или `https://t.me/c/<id>/12345`
для приватных чатов, с топиком — `.../thread/12345`), если Telegram
разрешает построить ссылку. Для чатов без username и с положительным
chat_id (личные переписки) ссылки не генерируются — пресет подставляет
просто `#12345`.

## Формат файла

```markdown
---
name: summary                   # должно совпадать с именем файла
prompt_version: v2              # бамп инвалидирует analysis_cache
needs_reduce: true              # нужна ли reduce-стадия для map-reduce
filter_model: gpt-5.4-nano      # модель для map-фазы (чанков)
final_model: gpt-5.4            # модель для финального ответа / single-chunk
output_budget_tokens: 1500      # max токенов на финальный ответ (reduce)
map_output_tokens: 1500         # max токенов на чанк в map-фазе (по умолчанию 1500)
---
Текст system-промпта. Задаёт роль модели.

---USER---

Текст user-промпта (инструкция). Должен содержать плейсхолдеры:
{period}, {title}, {msg_count}, {messages}.

Период: {period}
Чат: {title}
Сообщений: {msg_count}
---
{messages}
```

## Форум-чаты

Если анализируемый чат — форум, преамбула получает ссылку со вставкой
thread_id: `https://t.me/{chat}/{thread_id}/{msg_id}`. Пресет про это
знать не обязан: шаблон из преамбулы подставляется сам.

Для форума есть три режима (на уровне CLI):

- `--thread N` — один топик. Пресет видит только сообщения этого треда
  и цитирует их через стандартный `#msg_id`.
- `--all-flat` — весь форум «плоско». Пресет видит пересыпанные
  сообщения всех топиков; ссылки идут без `/thread/`, но всё равно
  ведут к конкретному сообщению.
- `--all-per-topic` — по пресету на каждый топик. Отчёты складываются
  в `reports/{chat-slug}/{topic-slug}-{preset}-{stamp}.md`.

## Что видит модель в `{messages}`

Формат сообщения в промпте:

```
[HH:MM #12345] sender_name [fwd: src] [voice 0:23]: ↩reply_sender text
```

- `#12345` — `msg_id`. Используй его в ссылках.
- В преамбуле выше есть строка `Ссылка на сообщение: https://t.me/...{msg_id}`
  — шаблон, куда подставить msg_id. Если её нет — значит чат не поддерживает
  t.me-ссылки (личные переписки без username), пиши просто `#12345`.

## Добавить свой пресет

1. Скопируй любой файл (например `summary.md` → `my_preset.md`).
2. Поменяй `name:` внутри на `my_preset`, отредактируй промпты.
3. Запусти: `uv run analyzetg analyze @chat --preset my_preset`.

Для одноразового промпта без регистрации файла используй
`--preset custom --prompt-file path/to/my.md` — тот же формат.

## Кэш

`prompt_version` в frontmatter — часть ключа `analysis_cache`. Пока он не
меняется, повторные запуски того же пресета на тех же сообщениях идут из
кэша (бесплатно). Бампай версию, когда меняешь промпты существенно.
